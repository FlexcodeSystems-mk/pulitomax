Frontend-only React + Vite SPA. Registered office: Viale Valganna 212, 21100 Varese (VA); P.IVA 04021430121; REA VA-396046; PEC ahmedehab@pec.it. Production domain: pulitomax.it.

## Run & Operate

- App runs via the `artifacts/cleaning: web` workflow (Vite dev server, binds to `PORT`)
- `pnpm --filter @workspace/cleaning run typecheck` ‚Äî typecheck the cleaning artifact
- No backend / no database ‚Äî all content is static; contact & quote forms submit client-side to FormSubmit (https://formsubmit.co/ajax/<email>), which emails submissions to the destination address (no API key)

## Stack

- React 19 + Vite 7, TypeScript
- Routing: wouter (locale-prefixed, single flat router ‚Äî no `nest`)
- i18n: custom React context (`src/lib/i18n.tsx`), locales it/en/de
- Forms: react-hook-form + zod (`@hookform/resolvers`)
- Animation: framer-motion
- Icons: lucide-react (brand/social icons also from lucide; `react-icons/si` for WhatsApp)
- Styling: Tailwind v4 (tokens via `hsl(var(--token))`), shadcn-style ui components

## SEO & Google Search Console

- `SITE.url` (in `src/lib/site.ts`) is the canonical production domain ‚Äî UPDATE it to the real live domain before deploy.
- `public/sitemap.xml` + `public/robots.txt` are generated by `pnpm --filter @workspace/cleaning run sitemap` (script: `scripts/generate-sitemap.mjs`). Re-run after adding pages/services/blog posts (route lists are hardcoded in that script ‚Äî keep them in sync).
- hreflang alternates (it/en/de + x-default) are injected per-page at runtime by the `Seo` component. Do NOT add static `<link rel="alternate" hreflang>` to `index.html` (causes duplicate/conflicting sets on non-home routes).
- Google Search Console verification: paste the `google-site-verification` meta tag into `index.html` (placeholder comment marks the spot), or verify via DNS / sitemap submission.

## Architecture decisions

- Locale routing uses a SINGLE flat wouter router (base = `BASE_URL`), NOT `nest`. `Router()` derives the locale from the first path segment; `LocalizedRouter` declares full locale-prefixed routes via `localePath(locale, ...)`. Do NOT reintroduce `nest`: links use absolute `localePath` hrefs (`/it/services`), and `nest` would prepend the locale base again ‚Üí doubled prefixes (`/it/it/services`) and 404s on click / language switch.
- The `I18nProvider` is keyed by locale (`key={locale}`) so it remounts on language change and the locale is fixed by the URL, not stored state.
- Language switcher (`Navbar`): swap only the leading locale segment of wouter's base-stripped `useLocation()` location: `location.replace(/^\/[a-z]{2}(?=\/|$)/, "")`, then `navigate(\`/${next}${rest}\`)`.
- Forms submit client-side to FormSubmit (`QuoteForm.tsx`; endpoint = `FORM_ENDPOINT` in `site.ts`). No API key/secret. On success they show a polished success state; on failure they show an inline error block (with a "retry" button) plus WhatsApp/phone fallback. FormSubmit success JSON is `{ success: "true" }` (string). Activation: the FIRST submission to a new address triggers a one-time confirmation email to the destination ‚Äî the recipient must click the link once before submissions deliver. To hide the personal email from scrapers, swap `FORM_ENDPOINT` to a FormSubmit alias (random string from the activation email). NOTE: we previously tried Web3Forms but its endpoint is behind a Cloudflare bot challenge that blocks browser `fetch()` (can't solve JS challenge) ‚Äî do not switch back.
- Per-page SEO is handled by the `Seo` component (title/meta/OG + JSON-LD injected into `<head>`).

## Gotchas

- Locale routing is a single flat wouter router ‚Äî never reintroduce `nest` (links use absolute `localePath` hrefs, so `nest` doubles the locale prefix ‚Üí `/it/it/...` and 404s on click).
- `react-icons/si` does not export `SiLinkedin` ‚Äî use lucide `Linkedin` for social icons.
- Verify with `pnpm --filter @workspace/cleaning run typecheck`, not `build` (build needs workflow-provided env).
